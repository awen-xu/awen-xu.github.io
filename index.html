C# Windows Forms - Calculator

Here is a simple calculator app that has a standard mode and a quadratic equation mode. The standard mode handles common operations, such as addition, subtraction, multiplication, division, and more. The quadratic mode calculates the roots of a quadratic equation. The user inputs the coefficients needed in the quadratic equation.

SQL - Mock Health Store

For my Database Concepts course, I have created the structure and the tables needed for a health store that sells supplements. I developed the entity-relationship diagram, and implemented the relationships in SQL developer using primary key and foreign key constraints. In addition, I also included other constraints to make sure that certain columns can't be null or can only hold a set of values.

Software Design - FoodStacks

For my Software Engineering Methodologies course, I have worked in a team with 4 other students to design a project, in which a food catering business called FoodStacks is planning to expand their business by adding an online ordering system. As a team, we established the function and non functional requirements and developed UML diagrams, such as use case diagrams, class diagrams, state machine diagrams, sequence diagrams, and more.

Unity Game - Fortissimo

Fortissimo is a simple 2D top down shooter game that I made in high school. It was made for an assignment from my physics class. This game was used to teach younger students about a specific job related to the study of physics. In this case, the particular specification is an acoustic engineer and how its study is used in making the LRAD (Long Range Acoustic Device), which is a tool for crowd/riot control used by authorities. I used Unity to write all the codes. As there were no lessons about programming or coding in my physics class, I had to find resources online in order to learn how to write the code.

TicTacToe – Java

I made a Tic Tac Toe game through Java RTP in grade 11. It is simple in design. There are two players, and each takes turn to enter a number that determines the placement on the board. The game ends when either a player wins, or neither win after all tiles are filled. Each tile is given a number ID, and the players place their markers on a specific tile by entering the number associated with that tile. If statements are used to determine the outcome of the game.

Battleship – Java

In grade 12, I wrote a little program that allows two players to play battleship. The game starts with having player 1 set down their ships on the tiles they want. Then, player 2 does the same. After the ship placement, the players take turn at guessing were their opponent’s ships are by clicking on the tiles. If they guess correctly, there will be an 'X' indicating that they have hit a ship. There is a status bar on the bottom that informs the players on the current standing of the game. The game ends when all ship pieces of one player are found.
